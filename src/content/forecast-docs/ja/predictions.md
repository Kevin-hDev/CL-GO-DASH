# 予測

予測とは、Forecast がターゲットの今後の値を見積もるために計算した結果です。「過去のデータと既知のコンテキストが一貫していれば、次にどのような値が期待できるか」という具体的な問いに答えます。

## 予測が表すもの

予測には、将来のポイントの連なりが含まれます。

各ポイントは日付または期間に対応します：

```text
2026-06-01 -> 142 commandes 予測
2026-06-02 -> 151 commandes 予測
2026-06-03 -> 149 commandes 予測
```

これらの値は確定ではありません。モデルの推定値を表します。

## 必要な入力

予測を開始するために、Forecast には以下が必要です：

| 要素 | 役割 |
| --- | --- |
| 日付 | 各行を時間軸に配置する |
| ターゲット | 予測対象の値 |
| 頻度 | データのリズム：日、時間、月など |
| ホライズン | 予測する将来ポイントの数 |
| モデル | 軌跡を計算するエンジン |

コンテキスト変数とマルチシリーズは必須ではありませんが、未来を説明またはシミュレーションする段階で重要になります。

## ホライズン

ホライズンは予測の深さを示します。

例：

- 1 時間頻度でホライズン `24`：今後 24 時間を予測；
- 日次頻度でホライズン `31`：今後 31 日を予測；
- 月次頻度でホライズン `12`：今後 12 か月を予測。

ホライズンが長いほど、一般的に不確実性は大きくなります。

## 結果と識別子

各実行は `analysis_id` 識別子を生成します。

この識別子は「保存されたファイル」を意味するわけではありません。計算された結果（将来の曲線、不確実性、パラメータ、変数、シナリオ、注釈）を取り出すために使われます。

アプリケーションはこれを使って：

- 予測を再び開く；
- グラフを表示する；
- 複数の結果を比較する；
- シナリオを作成または再実行する；
- LLM エージェントに結果を読み取らせる。

## 正しい解釈

予測は 3 つの問いで読む必要があります：

- 傾向は上昇、下降、安定のどれか？
- 不確実性は小さいか、大きいか？
- どのコンテキスト変数がその動きを説明できるか？

曲線だけでは不十分です。予測がそのコンテキストと結びついたとき、Forecast は有用になります。

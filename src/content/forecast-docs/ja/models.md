# モデル

モデルは、予測を計算するエンジンです。Forecast は複数のモデルファミリーを提供します。ローカル（計算がマシン上で完結）またはクラウド（必要なデータが設定されたプロバイダーに送信される）のいずれかです。

## ローカルファミリー

| ファミリー | 提供元 | 詳細 |
| --- | --- | --- |
| Chronos / Chronos-Bolt | Amazon | 高速なローカルモデル。最初のテストやシンプルなターゲットに適しています |
| TimesFM | Google | 時系列予測のローカルモデル |
| Toto 2.0 | Datadog | モニタリングとメトリクス向けのローカルモデル |
| MOIRAI 2.0 | Salesforce | マルチシリーズと共変量を扱うローカルモデル |
| FlowState | IBM | 時系列向けのローカルモデル |
| TabPFN-TS | PriorLabs | 実験的なローカルモデル |
| TiRex | NX-AI | 実験的なローカルモデル |
| Kairos | Foundation Model Research | 実験的なローカルモデル |
| Sundial | THUML | 実験的なローカルモデル |

## クラウドファミリー

| ファミリー | 提供元 | 詳細 |
| --- | --- | --- |
| TimeGPT-2 / TimeGPT-2.1 | Nixtla | 時系列特化のクラウドエンジン。API キーが必要で、必要なデータをプロバイダーに送信します。 |

クラウドモデルはより強力な場合がありますが、外部依存とデータ送信を伴います。機密データの場合は、ローカルモデルを優先してください。

## モデルの選び方

選択は主にデータセットとユースケースに依存します：

- **手軽なテスト、シンプルなターゲット**：Chronos-Bolt。
- **機密データ、ローカル計算**：任意のローカルファミリー。
- **共変量と既知の未来**：コンテキスト変数を扱うモデル（MOIRAI 2.0、Chronos-2、TimeGPT）。
- **マルチシリーズ**：複数系列を扱うモデル（MOIRAI 2.0、Chronos-2、TimeGPT）。
- **高度なクラウド品質**：TimeGPT（データ送信を受け入れる場合）。

高度なモデルは、不適切に構造化されたデータを補うことはできません。モデルを変える前に、データセットの品質、頻度、ホライズン、コンテキスト変数を確認してください。

## ローカルモデルのインストール

ローカルモデルは、モデルマネージャー（Settings → Forecast）または Forecast ワークスペースのモデルタブからインストールする必要があります。ファミリーに応じて Hugging Face または GitHub からダウンロードされ、ローカルの `~/.local/share/cl-go-dash/forecast-models/` に保存されます。

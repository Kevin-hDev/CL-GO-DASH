# 共変量

共変量とは、ターゲットに影響を与える可能性のあるコンテキスト変数です。予測対象のデータの周囲の環境を記述します。

## 定義

共変量は次の問いに答えます。「ターゲットがなぜ上がる、下がる、リズムが変わるのかをモデルが理解するのに、何が役立つか？」

例：

| 分野 | ターゲット | 可能な共変量 |
| --- | --- | --- |
| レストラン | 命令数 | 天気、祝日、地域イベント、プロモ |
| 金融 | 価格または取引量 | VIX、金利、ニューススコア、ビットコイン、ドル指数 |
| SaaS | サーバー負荷 | アクティブユーザー、キャンペーン、障害、リリース |
| 小売 | 売上 | 在庫、値引き、季節、店舗トラフィック |

## なぜ重要か

共変量がないと、モデルは主にターゲットの過去を見ます。

共変量があれば、モデルはコンテキストも利用できます：

- 「プロモが有効なとき売上はしばしば上がる」；
- 「大雨のとき命令数は下がる」；
- 「VIX が上がるとき価格が反応する」；
- 「マーケティングキャンペーン後に負荷が増える」。

つまり、過去を単に延長しただけの予測よりも、コンテキストを踏まえた予測が可能になります。

## コンテキスト変数の型

共変量は次のいずれかに分類されます：

| 型 | 例 | 読み方 |
| --- | --- | --- |
| 数値 | `temperature = 28` | 測定値 |
| パーセンテージ | `discount_pct = 15` | 効果の強さ |
| バイナリ | `weekend = 1` | はい / いいえ |
| イベント | `concert_local = 1` | イベントあり |
| スコア | `news_score = 0.72` | 計算された指標 |
| カレンダー | `jour_ferie = 1` | 時間的なコンテキスト |

## 履歴と既知の未来

共変量は 2 つのゾーンに存在するとき、より有用になります：

- 履歴：モデルはその変数がターゲットにどう影響したかを学習します；
- 既知の未来：モデルはその将来の値を使ってターゲットを予測します。

例：

```text
date        commandes   pluie_mm   weekend
2026-05-01  120        0          0
2026-05-02  148        4          1
2026-05-03             12         1
```

将来のターゲットは空ですが、`pluie_mm` と `weekend` は既知です。モデルはこれらの将来の条件を考慮して予測できます。

## シナリオ内の変数

コンテキストシナリオでは、ユーザーが将来の共変量を変更します。

例：

- `vix_close` を 20% 増加させる；
- 1 週間 `promo_active` を 1 にする；
- `temperature` を 5 度下げる；
- 高い `breach_alert_level` をシミュレーションする；
- 将来の日付の `trafic_indice` を変更する。

Forecast はこの新しいコンテキストでモデルを再実行し、新たな軌跡を生成します。

## 金融変数辞書

金融データセットでは、コンテキスト変数に専門用語が使われることがあります。この表は、金融シナリオで表示される変数を説明します。

| 変数 | 表すもの | 簡単な読み方 |
| --- | --- | --- |
| `nasdaq_return_pct` | ナスダックのパーセント変動 | テック市場の上昇・下落を示す |
| `vix_close` | 終値の VIX レベル | 市場の恐怖やボラティリティを示す |
| `btc_close_usd` | ビットコインのドル価格 | リスクシグナルや投機的アペタイトとして機能 |
| `usd_index_dxy` | 米ドル指数 | 他通貨に対するドルの強さを示す |
| `treasury_10y_pct` | 米 10 年国債金利 | 長期の資金調達コストを表す |
| `sector_etf_volume_musd` | セクター ETF の取引量（百万ドル） | 特定セクターの活動を示す |
| `breach_alert_level` | 脆弱性やサイバー事故のアラートレベル | サイバーセキュリティセクター特有のストレスを表す |
| `zero_day_news_score` | ゼロデイ脆弱性に関するニューススコア | 重大なサイバーニュースの強さを示す |
| `gov_contract_flow_index` | 公的契約のフロー指数 | 政府契約の勢いを表す |
| `earnings_heat_index` | 決算発表前後の緊張度指数 | 決算期間の重要性や感応度を示す |
| `ai_capex_signal` | AI 関連支出の投資シグナル | AI 投資テーマの強さを表す |
| `fed_event_flag` | 米中央銀行のイベント指標 | Fed イベントがある日はしばしば 1 になる |
| `option_expiry_flag` | オプション満了指標 | オプションが値動きに影響しうる日を示す |
| `month_end_flag` | 月末指標 | 月次リバランスやクロージングの効果を示す |
| `weekend` | 週末指標 | 主にカレンダー効果のある日次データで使用 |

## これらの変数の読み方

これらの変数は単独で予測するものではありません。モデルにコンテキストを与えます。

例：

- `vix_close` が上がれば、市場はしばしばより神経質になります；
- `fed_event_flag` が 1 なら、その日は金利発表に反応しやすくなります；
- `zero_day_news_score` が上がれば、サイバー関連銘柄が反応することがあります；
- `sector_etf_volume_musd` が上がれば、セクターの活動がより強いことを示します；
- `month_end_flag` が 1 なら、月末クロージングによる値動きがある可能性があります。

モデルは履歴の中で、これらの変数がターゲットの動きとともに現れたかどうかを学習します。

## シナリオでの使用

シナリオタブでこれらの変数を変更することは、仮説を立てることと同じです。

例：

| 仮説 | 可能な変更 |
| --- | --- |
| よりストレスの強い市場 | `vix_close` +20% |
| より強いドル | `usd_index_dxy` +2% |
| Fed の日 | `fed_event_flag` = 1 |
| 強いサイバーニュース | `zero_day_news_score` +30% |
| 影響の大きい月末 | `month_end_flag` = 1 |

再実行後、Forecast はこの新しい将来コンテキストで軌跡を再計算します。

## 避けるべき落とし穴

共変量は準備を誤ると結果を悪化させることがあります。

避けるべきこと：

- 既知の未来で変数が空；
- 情報を何も持たない定数変数；
- 活用可能な値に変換されていない自由テキスト；
- 間接的に将来のターゲットを含む変数；
- 同じ列に混在した単位；
- 明確な仮説なしにでっち上げられた将来の値。

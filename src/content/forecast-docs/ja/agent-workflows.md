# LLM エージェント

LLM エージェントは Forecast を専門エンジンとして使えます。その役割は、ファイルを読んでツールをクリックするだけにとどまりません。データの準備、ウェブでのコンテキスト検索、データセットの構築、予測の実行、結果の説明まで行えます。

## エージェントにできること

エージェントは複数の段階で介入できます：

| 段階 | エージェントの役割 |
| --- | --- |
| 準備 | Excel、CSV、JSON を読み込む |
| 検索 | ウェブで外部情報を収集する |
| データセット | 有用な列を作成または補強する |
| 実行 | 適切なパラメータで `forecast` を呼び出す |
| 読み取り | `forecast_read` で結果を取り出す |
| シナリオ | 仮説を作り、モデルを再実行する |
| 説明 | 傾向、不確実性、変数、限界を要約する |

例：金融予測では、エージェントはローカルファイルを読み、最近の市場コンテキストを調べ、`news_score` や `event_flag` のような列を生成してから Forecast を起動できます。

## 推奨ワークフロー

エージェントはこの順序に従う必要があります：

1. ユーザーの要求を理解する；
2. 利用可能なデータを調べる；
3. 予測対象のターゲットを特定する；
4. 日付、頻度、ホライズンを特定する；
5. 必要に応じて有用なコンテキスト変数を探す、または作成する；
6. 未来の行が一貫しているか確認する；
7. 互換性のあるモデルを選択する；
8. `forecast` を実行する；
9. `forecast_read` で結果を読み取る；
10. 予測を説明し、有用なシナリオを提案する。

## エージェントによるデータ作成

エージェントは、ユーザーの要請や予測の必要に応じてデータを作成できます。

例：

- 日付から `weekend` 列を追加する；
- `month_end_flag` を作成する；
- ウェブのイベントを数値スコアに変換する；
- 仮定の天気で未来ゾーンを埋める；
- ワークフローを検証するためのテストデータセットを作成する；
- Excel ファイルを利用可能な JSON に変換する。

エージェントは、どの列を作成し、なぜ作成したかを常に説明する必要があります。

## 安全性と品質のルール

エージェントは、重要なデータを暗黙にでっち上げてはなりません。

変数を作成する場合、以下を区別する必要があります：

- ファイルから読み取ったデータ；
- ウェブで見つけたデータ；
- 計算で作成したデータ；
- シミュレーションの仮定。

この区別は、ユーザーが何が実データで、何が計算または仮定かを把握するために不可欠です。

## スラッシュコマンド

スラッシュコマンドは、エージェントとユーザー向けのクイックガイドとして機能します。

例：

- `/forecast`：Forecast モジュールを理解する；
- `/forecast-predict`：予測を準備して実行する；
- `/forecast-dataset`：きれいなデータセットを構築する；
- `/forecast-scenarios`：有用な仮説を作成する；
- `/forecast-cmd`：利用可能なツールを理解する。

これらのコマンドは、短く、明確で、すぐに実行可能な手順を提供する必要があります。

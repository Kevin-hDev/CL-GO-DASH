# Forecast ツール

Forecast ツールを使うと、LLM エージェントがチャットから予測エンジンを利用できます。これらは正確なパラメータで呼び出す必要があります。列の指定ミスや一貫性のないホライズンは、無意味な予測を生むからです。

## `forecast`

`forecast` は新しい予測を開始します。

主な入力：

| パラメータ | 役割 |
| --- | --- |
| `file_path` | 読み込む Excel、CSV、JSON ファイル |
| `data` | すでに準備済みの JSON データ |
| `date_column` | 日付を含む列 |
| `target_column` | 予測対象の列 |
| `series_column` | 系列を識別する列 |
| `covariate_columns` | 使用するコンテキスト変数 |
| `frequency` | 時間のリズム |
| `horizon` | 将来ポイントの数 |
| `model` | 使用するエンジン |

主な出力：

- `analysis_id`、Forecast 結果の識別子。

## `forecast_read`

`forecast_read` は Forecast の結果を読み取ります。

以下を取り出すために使います：

- 予測；
- 履歴；
- 不確実性；
- シナリオ；
- 利用可能な変数；
- モデルのメタデータ。

`analysis_id` が渡されない場合、エージェントはこれを使って利用可能な結果の一覧を取得できます。

## `forecast_analyze`

`forecast_analyze` は予測周辺の要素を追加または変更します。

特に以下に使われます：

- 注釈の作成；
- シナリオの作成；
- コンテキストシナリオの再実行；
- シナリオの変更；
- シナリオの削除。

## エージェントが確認すべきこと

ツールを呼び出す前に、エージェントは以下を確認する必要があります：

- ターゲットが存在する；
- 日付が読み取り可能；
- ホライズンが将来の行と一致する；
- 共変量が実際に存在する；
- 作成またはウェブで見つけたデータが識別されている；
- 選択したモデルが要件をサポートする。

エージェントは、不透明な呼び出しを行うのではなく、自分の選択を説明する必要があります。

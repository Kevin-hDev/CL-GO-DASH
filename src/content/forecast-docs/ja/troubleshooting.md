# 診断

このセクションは、予測が期待通りの結果を出さない理由を理解するのに役立ちます。

## 無効な JSON データ

このエラーは、Forecast が利用可能な表を受け取れなかったことを意味します。

原因としては：

- 不正な形式の JSON；
- 正しく変換されなかったファイル；
- 空または切り詰められた `data` フィールド；
- 行フォーマットの誤り；
- 列の欠落。

ユーザーがファイルを提供した場合、エージェントはデータを変換する前にファイルが正しく読み込まれているか確認する必要があります。

## モデルが利用できない

このエラーの原因としては：

- ローカルモデルが未インストール；
- サイドカーが停止している；
- API キーがない；
- パラメータと互換性のないモデル；
- 要求されたモデルに対してデータが短すぎる。

正しい対応は、モデルを確認してから最小のデータセットでテストすることです。

## コンテキスト変数が無視される

変数は次の場合に無視されたり無意味になったりすることがあります：

- 履歴に存在しない；
- 未来で空である；
- 定数である；
- 型が不正；
- ホライズンに合わない；
- 数値やカテゴリに変換されていないテキストを含む。

この場合、モデルを責める前にデータセットを調べる必要があります。

## 平坦な結果

平坦な予測は、ターゲットが安定しているなら正常です。

また以下を示すこともあります：

- 履歴が短すぎる；
- 頻度の選択が誤っている；
- コンテキストがない；
- ターゲットの変動が少ない；
- モデルが単純すぎる；
- 既知の未来の情報量が少ない。

## 効果が見えないシナリオ

コンテキストシナリオがほとんど差を示さないことがあります：

- 変更した変数の影響が小さい；
- 変更が小さすぎる；
- モデルがその変数を強いシグナルとして使わない；
- 将来の変数が実際に渡されていない；
- フィルターで曲線が隠れている。

グラフ、フィルター、ツールチップ、シナリオのデータを確認する必要があります。

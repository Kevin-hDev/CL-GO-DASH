# 不確実性

予測が絶対的な真実になることはありません。そのため Forecast は、軌跡周辺のリスクを示すため、中心値と不確実性の区間を表示します。

## 中心値

中心値はモデルの主な推定値です。

例：

```text
2026-06-01 -> 142 commandes 予測
```

これは使用されたデータに基づく最も確からしいシナリオを表します。

## 不確実性の区間

区間は、中心値周辺の確からしい範囲を示します。

例：

```text
予測値：142
下限：128
上限：157
```

簡単な読み方：モデルは 142 と推定していますが、128 から 157 程度の値なら妥当とみなします。

## 分位数

モデルは分位数を返すことがあります。

| フィールド | 意味 |
| --- | --- |
| q10 | 確からしい低い値 |
| q50 | 中心値または中央値 |
| q90 | 確からしい高い値 |

q10 と q90 の差が広いほど、モデルの不確実性が大きくなります。

## 不確実性が増す理由

次の場合に不確実性は増すことがあります：

- 履歴が短い；
- ターゲットの変動が激しい；
- ホライズンが長い；
- コンテキスト変数がない；
- データに最近の急変が現れた；
- 複数の将来シナリオが可能である。

## 適切な使い方

中心値は傾向を読むために使います。

不確実性の区間はリスクを読むために使います。

重大な意思決定では、両方を確認する必要があります。

# マルチシリーズ

マルチシリーズを使うと、1 つの分析で複数の系列を予測できます。各対象ごとに別々の予測を実行する代わりに、Forecast は各系列を識別する列を持つ 1 つのデータセットを受け取ります。

## 例

1 つのファイルに複数店舗の売上が含まれることがあります：

```text
date        magasin   ventes   promo
2026-05-01  paris    120      0
2026-05-01  lyon     92       1
2026-05-02  paris    132      1
2026-05-02  lyon     95       0
```

ここでは `magasin` が系列列です。

Forecast は `paris` と `lyon` の両方の売上を予測すると理解します。

## 何の役に立つか

マルチシリーズは、複数の系列が共通のロジックを持つ場合に有用です。

例：

- 店舗ごとの売上；
- レストランごとの命令数；
- サーバーごとのトラフィック；
- 資産ごとの価格；
- 地域ごとのインシデント数。

モデルは、系列が似ている、またはコンテキスト変数を共有している場合に特に、個別予測よりも多くの情報を活用できます。

## 系列ごとのホライズン

各系列は一貫した時間構造を提供する必要があります。

ホライズンが `31` であれば、各系列に 31 個の予測対象となる将来ポイントが必要です。

例：

```text
paris -> 31 行の未来データ
lyon  -> 31 行の未来データ
```

一貫性のないホライズンは比較を難しくし、モデルを止めることがあります。

## Forecast での読み方

インターフェースでは、ユーザーが表示する系列を選択できます。

その後、シナリオを次のように読めます：

- 特定の系列；
- 複数の系列；
- ベースライン予測との比較。

マルチシリーズは Forecast の原則を変えません。単に「いまどの系列が予測されているか」という次元を追加するだけです。

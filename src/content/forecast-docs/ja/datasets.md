# データセット

データセットとは、Forecast が過去を学習し未来を予測するために使用する表です。モデルが何を、どの時点で、どのコンテキストで予測すべきか理解できるよう、構造化されている必要があります。

## 最小構成

Forecast のデータセットには、最低限以下が含まれます：

| 列 | 役割 |
| --- | --- |
| Date | 各観測がいつ発生したかを示す |
| ターゲット | 予測対象の値 |

簡単な例：

```text
date        commandes
2026-05-01  120
2026-05-02  135
2026-05-03  128
```

この表があれば、Forecast は commandes の動きを学習し、今後の日付を予測できます。

## 履歴ゾーン

履歴ゾーンには、すでに分かっている実績値が含まれます。

モデルはここから以下を検出します：

- 傾向；
- 季節性；
- リズム；
- ピーク；
- 落ち込み；
- 通常の変動。

このゾーンではターゲットが入力されている必要があります。

## 未来ゾーン

未来ゾーンには、予測対象の日付が含まれます。

このゾーンではターゲットは空です。それこそが Forecast が計算すべきものだからです。

例：

```text
date        commandes
2026-05-01  120
2026-05-02  135
2026-05-03
2026-05-04
```

ここでは、Forecast が 5 月 3 日と 4 日の commandes を予測します。

## 既知の未来

既知の未来は、未来の行にコンテキスト変数を追加します。

例：

```text
date        commandes   pluie_mm   promo
2026-05-01  120        0          0
2026-05-02  135        4          1
2026-05-03             12         0
2026-05-04             0          1
```

将来のターゲットは空ですが、pluie_mm と promo は既知、または想定された値です。モデルはこれらの情報を使って、より現実的な予測を生成できます。

## 系列列

系列列は、同じファイルに複数の予測対象が含まれる場合に使います。

例：

- 複数の店舗；
- 複数の商品；
- 複数の都市；
- 複数の金融資産；
- 複数のサーバー。

例：

```text
date        magasin   ventes   promo
2026-05-01  paris    120      0
2026-05-01  lyon     92       1
2026-05-02  paris    132      1
2026-05-02  lyon     95       0
```

これにより、Forecast は各系列が属するグループを考慮しながら予測できます。

## エージェントが作成したデータセット

LLM エージェントは、データセットを作成または補強できます。

例えば：

- Excel を JSON に変換する；
- `weekend` 列を追加する；
- ウェブでイベント情報を収集する；
- テキスト情報をスコアに変換する；
- 仮説で未来の行を埋める；
- 日付や列をクリーンアップする。

エージェントは、どのデータがファイル・ウェブ・計算・仮説からのものかを明確に示す必要があります。

# 概要

Forecast は、測定可能なデータの将来の動きを予測するために使用します。このモジュールは、履歴・最近の傾向・コンテキスト変数を分析し、不確実性の幅と比較可能なシナリオを伴う日付付きの予測を生成します。

## 予測の簡単な定義

予測とは、過去および現在のデータを観察し、次に何が起こるかを見積もることです。

例：

- 今後 30 日間の売上を予測する；
- 来月の売上高を見積もる；
- 今後数時間のサーバー負荷を想定する；
- 資産の価格や取引量を予測する；
- 既知の将来のコンテキストの影響をシミュレーションする。

モデルは未来を読むわけではありません。データに見えるパターンから確からしい軌跡を計算します。

## Forecast が LLM チャットに追加するもの

LLM は表を読み、説明を書くことができます。Forecast は、実際に将来の数値列を計算する専門エンジンを追加します。

この違いは重要です：

| LLM チャット単体 | Forecast |
| --- | --- |
| ファイルを説明する | 将来の日付付きポイントを計算する |
| 定性的に推論できる | 数値の曲線を出力する |
| データが曖昧だとでっち上げることがある | 厳密なデータ契約を使用する |
| 傾向を要約する | 予測・区間・シナリオを生成する |

LLM はエンジンの周辺で有用であり続けます。データを準備し、列を選択し、ウェブで情報を探し、データセットを構築し、Forecast を起動して、結果を説明します。

## 主な対象：ターゲット

ターゲットとは、Forecast が予測すべき列のことです。

例：

- `ventes`
- `ca_total_eur`
- `commandes_total`
- `temperature`
- `stock_price`
- `incidents_count`

予測はすべてこのターゲットを中心に展開します。モデルは過去の振る舞いを学習し、将来の値を見積もります。

## Forecast の結果に含まれるもの

Forecast の結果には以下が含まれます：

- 使用された履歴値；
- ポイントごとの将来予測；
- 不確実性の区間；
- 使用されたコンテキスト変数；
- この予測から作成されたシナリオ；
- 結果を再確認・比較・エクスポートするためのメタデータ。

これは受動的なファイルではありません。何が確からしく、何がリスクであり、コンテキストが変化すれば何が変わるかを理解するための完全な作業オブジェクトです。

## 全体的なロジック

標準的なワークフローは次のとおりです：

1. データセットを提供する；
2. 日付、ターゲット、必要に応じて系列を選択する；
3. 有用なコンテキスト変数を選択する；
4. 予測モデルを起動する；
5. 将来の曲線と不確実性を読み取る；
6. 仮説を検証するためのシナリオを作成する；
7. LLM に結果の説明や新しいデータの準備を依頼する。

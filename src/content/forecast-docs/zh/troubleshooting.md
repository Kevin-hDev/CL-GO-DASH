# 诊断

这一节帮助你理解为什么一份预测没有得到预期的结果。

## JSON 数据无效

这个错误意味着 Forecast 没有收到一个可用的表格。

原因可能是：

- JSON 格式错误；
- 文件未被正确转换；
- `data` 字段为空或被截断；
- 行格式错误；
- 缺少列。

如果用户提供的是文件，代理应先确认文件被正确读取，再进行数据转换。

## 模型不可用

这个错误可能源于：

- 本地模型未安装；
- sidecar 已停止；
- 缺少 API 密钥；
- 模型与参数不兼容；
- 相对于所请求的模型，数据过短。

正确的做法是先检查模型，然后用一个最小数据集进行测试。

## 上下文变量被忽略

某个变量可能被忽略或不起作用，如果：

- 它在历史中不存在；
- 它在未来中为空；
- 它是常量；
- 它的类型不对；
- 它与预测期不匹配；
- 它包含未转换为可用数字或类别的文本。

这种情况下，在归咎于模型之前，应先检查数据集。

## 平淡的结果

如果目标本身稳定，平淡的预测可能是正常的。

它也可能表明：

- 历史过短；
- 频率选择不当；
- 缺少上下文；
- 目标本身变化不大；
- 模型过于简单；
- 已知未来的信息量不足。

## 场景没有可见效果

上下文场景可能几乎看不出差异，如果：

- 被修改的变量影响很小；
- 修改幅度太小；
- 模型并未把该变量当作强信号；
- 未来变量实际并未被传入；
- 曲线被过滤器隐藏。

应检查图表、过滤器、提示信息和场景数据。

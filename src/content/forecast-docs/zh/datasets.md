# 数据集

数据集是 Forecast 用来学习过去并预测未来的表格。它的结构必须能让模型理解要预测什么、在哪些日期、以及带有什么上下文。

## 最小结构

一个 Forecast 数据集至少包含：

| 列 | 作用 |
| --- | --- |
| 日期 | 表示每次观察发生的时间 |
| 目标 | 需要预测的值 |

简单示例：

```text
date        commandes
2026-05-01  120
2026-05-02  135
2026-05-03  128
```

有了这张表，Forecast 就能学习 commandes 的动态，并预测接下来的日期。

## 历史区

历史区包含已经已知的真实值。

它用于让模型检测：

- 趋势；
- 季节性；
- 节奏；
- 峰值；
- 下跌；
- 正常波动。

在这一区域，目标必须填写完整。

## 未来区

未来区包含需要预测的日期。

在这一区域，目标是空的，因为这正是 Forecast 需要计算的内容。

示例：

```text
date        commandes
2026-05-01  120
2026-05-02  135
2026-05-03
2026-05-04
```

这里，Forecast 需要预测 5 月 3 日和 4 日的 commandes。

## 已知未来

已知未来在未来行上添加上下文变量。

示例：

```text
date        commandes   pluie_mm   promo
2026-05-01  120        0          0
2026-05-02  135        4          1
2026-05-03             12         0
2026-05-04             0          1
```

未来目标是空的，但降雨（pluie_mm）和促销（promo）已经已知或假定。模型可以利用这些信息，生成更贴近现实的预测。

## 序列列

当同一个文件包含多个待预测对象时，就要用到序列列。

示例：

- 多个门店；
- 多个产品；
- 多个城市；
- 多个金融资产；
- 多台服务器。

示例：

```text
date        magasin   ventes   promo
2026-05-01  paris    120      0
2026-05-01  lyon     92       1
2026-05-02  paris    132      1
2026-05-02  lyon     95       0
```

Forecast 随后可以预测每个序列，同时考虑它所属的分组。

## 由代理创建的数据集

LLM 代理可以创建或丰富数据集。

例如它可以：

- 将 Excel 转换成 JSON；
- 添加一列 `weekend`；
- 从网上抓取事件；
- 将文本信息转换为分数；
- 用假设值填充未来行；
- 清洗日期或列。

代理必须清楚说明哪些数据来自文件、网络、计算还是假设。

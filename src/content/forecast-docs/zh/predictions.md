# 预测

预测是 Forecast 计算出的结果，用于估计某个目标的未来值。它回答一个具体问题：“如果过去的数据和已知的上下文保持一致，接下来我们可以期待什么值？”

## 一份预测代表什么

一份预测包含一连串未来数据点。

每个数据点对应一个日期或时段：

```text
2026-06-01 -> 预测 142 单订单
2026-06-02 -> 预测 151 单订单
2026-06-03 -> 预测 149 单订单
```

这些值并非定论。它们代表模型的估计。

## 所需输入

要启动一份预测，Forecast 需要：

| 元素 | 作用 |
| --- | --- |
| 日期 | 将每行定位到时间轴上 |
| 目标 | 需要预测的值 |
| 频率 | 数据的节奏：日、小时、月等 |
| 预测期 | 需要预测的未来点数 |
| 模型 | 用于计算轨迹的引擎 |

上下文变量和多序列并非必需，但一旦需要解释或模拟未来，它们就变得很重要。

## 预测期

预测期表示预测的深度。

示例：

- 预测期 `24`、按小时：预测未来 24 小时；
- 预测期 `31`、按日：预测未来 31 天；
- 预测期 `12`、按月：预测未来 12 个月。

预测期越长，不确定性通常越大。

## 结果与标识符

每次启动都会生成一个 `analysis_id` 标识符。

这个标识符并不意味着“已保存文件”。它用于检索已计算的结果：未来曲线、不确定性、参数、变量、场景和注释。

应用程序用它来：

- 重新打开一份预测；
- 显示图表；
- 对比多个结果；
- 创建或重新运行场景；
- 让 LLM 代理复查结果。

## 正确解读

阅读一份预测时应带着三个问题：

- 趋势是上升、下降还是稳定？
- 不确定性是较小还是较大？
- 哪些上下文变量可能解释这一走势？

单看一条曲线是不够的。当预测与它的上下文相关联时，Forecast 才真正有用。

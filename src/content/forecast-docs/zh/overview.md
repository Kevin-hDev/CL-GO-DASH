# 概览

Forecast 用于预测可度量数据未来的演变。该模块分析历史数据、近期趋势和上下文变量，从而生成带有日期的预测，并附带不确定性的区间和可对比的场景。

## 预测的简单定义

预测就是通过观察过去和现在的数据，来估计接下来可能发生什么。

示例：

- 预测未来 30 天的销量；
- 估计下个月的营业额；
- 预判未来几小时的服务器负载；
- 推演某个资产的价格或成交量；
- 模拟某个已知未来上下文的影响。

模型并不能预知未来。它根据数据中可见的规律，计算出一条可能的轨迹。

## Forecast 相对于 LLM 聊天的增量

LLM 可以读取表格并写出一段解释。Forecast 则增加了一个专用引擎，真正计算出未来的序列。

差别很重要：

| 仅 LLM 聊天 | Forecast |
| --- | --- |
| 解释一个文件 | 计算带日期的未来数据点 |
| 可以做定性推理 | 输出一条数值曲线 |
| 数据模糊时可能凭空编造 | 使用严格的数据契约 |
| 概括趋势 | 生成预测、区间和场景 |

LLM 在引擎周边仍然有用：准备数据、选择列、在网上搜索信息、构建数据集、启动 Forecast，然后解释结果。

## 核心对象：目标

目标是 Forecast 需要预测的那一列。

示例：

- `ventes`
- `ca_total_eur`
- `commandes_total`
- `temperature`
- `stock_price`
- `incidents_count`

整份预测都围绕这个目标展开：模型学习它过去的行为，然后估计它未来的值。

## Forecast 结果中包含什么

一个 Forecast 结果包含：

- 使用过的历史值；
- 逐点的未来预测；
- 不确定性区间；
- 使用过的上下文变量；
- 基于该预测创建的场景；
- 复查、对比和导出结果所需的元数据。

它不是一个被动的文件。它是一个完整的工作对象，用于理解什么是大概率的、什么是高风险的、以及上下文变化时什么会改变。

## 总体逻辑

标准工作流是：

1. 提供一个数据集；
2. 选择日期、目标，必要时还有序列；
3. 选择有用的上下文变量；
4. 启动一个预测模型；
5. 阅读未来曲线和不确定性；
6. 创建场景来测试假设；
7. 让 LLM 解释结果或准备新数据。

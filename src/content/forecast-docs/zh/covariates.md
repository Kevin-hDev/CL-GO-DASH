# 协变量

协变量是可能影响目标的上下文变量。它们描述待预测数据所处的环境。

## 定义

一个协变量回答的问题是：“什么能帮助模型理解目标为什么上升、下降或改变节奏？”

示例：

| 领域 | 目标 | 可能的协变量 |
| --- | --- | --- |
| 餐饮 | 订单 | 天气、节假日、本地活动、促销 |
| 金融 | 价格或成交量 | VIX、利率、新闻得分、比特币、美元指数 |
| SaaS | 服务器负载 | 活跃用户、活动、故障、发版 |
| 零售 | 销量 | 库存、折扣、季节、门店客流 |

## 为什么它们很重要

没有协变量时，模型主要只看目标的过去。

有了协变量，模型还可以利用上下文：

- “促销开启时销量往往会上升”；
- “大雨时订单会下降”；
- “VIX 上升时价格会做出反应”；
- “营销活动后负载会增加”。

因此，它们能生成比单纯延续过去更贴合上下文的预测。

## 上下文变量的类型

协变量可以是：

| 类型 | 示例 | 解读 |
| --- | --- | --- |
| 数值 | `temperature = 28` | 测量值 |
| 百分比 | `discount_pct = 15` | 某种效应的强度 |
| 二值 | `weekend = 1` | 是 / 否 |
| 事件 | `concert_local = 1` | 存在某事件 |
| 得分 | `news_score = 0.72` | 计算得出的指标 |
| 日历 | `jour_ferie = 1` | 时间上下文 |

## 历史与已知未来

如果某个协变量同时存在于两个区域，它会更有用：

- 历史：模型学习它如何影响目标；
- 已知未来：模型使用它的未来值来预测目标。

示例：

```text
date        commandes   pluie_mm   weekend
2026-05-01  120        0          0
2026-05-02  148        4          1
2026-05-03             12         1
```

未来目标是空的，但 `pluie_mm` 和 `weekend` 已知。因此模型可以在考虑这些未来条件的情况下进行预测。

## 场景中的变量

在上下文场景中，用户修改未来的协变量。

示例：

- 将 `vix_close` 提高 20%；
- 让 `promo_active` 在一周内变为 1；
- 让 `temperature` 下降 5 度；
- 模拟一个高位的 `breach_alert_level`；
- 改变未来若干天的 `trafic_indice`。

Forecast 随后用这一新上下文重新运行模型，生成一条新的轨迹。

## 金融变量词典

在金融数据集中，上下文变量可能带有技术性名称。下表解释了金融场景中可见的变量。

| 变量 | 代表什么 | 简单解读 |
| --- | --- | --- |
| `nasdaq_return_pct` | 纳斯达克的百分比变动 | 衡量科技市场是涨是跌 |
| `vix_close` | VIX 收盘水平 | 衡量市场的恐惧或波动率 |
| `btc_close_usd` | 以美元计的比特币价格 | 作为风险信号或投机偏好 |
| `usd_index_dxy` | 美元指数 | 衡量美元相对于其他货币的强弱 |
| `treasury_10y_pct` | 美国 10 年期国债收益率 | 代表长期资金成本 |
| `sector_etf_volume_musd` | 某行业 ETF 成交量（百万美元） | 衡量某个具体行业的活跃度 |
| `breach_alert_level` | 与漏洞或网络安全事件相关的警报级别 | 代表网络安全行业特有的压力 |
| `zero_day_news_score` | 围绕零日漏洞的新闻得分 | 衡量关键网络安全新闻的强度 |
| `gov_contract_flow_index` | 公共合同流量指数 | 代表政府合同的活跃度 |
| `earnings_heat_index` | 财报期紧张度指数 | 衡量财报时段的重要性或敏感度 |
| `ai_capex_signal` | 与 AI 支出相关的投资信号 | 代表 AI 投资主题的强度 |
| `fed_event_flag` | 美国央行事件指示 | 当存在 Fed 事件时常取值为 1 |
| `option_expiry_flag` | 期权到期指示 | 标记期权可能影响走势的交易日 |
| `month_end_flag` | 月末指示 | 标记再平衡或月度收盘效应 |
| `weekend` | 周末指示 | 主要用于带有日历效应的日度数据 |

## 如何解读这些变量

这些变量并不能单独做出预测。它们为模型提供上下文。

示例：

- 如果 `vix_close` 上升，市场往往更紧张；
- 如果 `fed_event_flag` 为 1，当天对利率公告可能更敏感；
- 如果 `zero_day_news_score` 上升，网络安全相关标的可能做出反应；
- 如果 `sector_etf_volume_musd` 上升，该行业活动更活跃；
- 如果 `month_end_flag` 为 1，某些走势可能来自月度收盘。

模型会在历史数据中学习这些变量是否曾伴随目标的走势。

## 在场景中使用

在“场景”标签页中修改这些变量，等同于提出一个假设。

示例：

| 假设 | 可能的修改 |
| --- | --- |
| 市场更承压 | `vix_close` +20% |
| 美元走强 | `usd_index_dxy` +2% |
| Fed 日 | `fed_event_flag` = 1 |
| 网络安全新闻火热 | `zero_day_news_score` +30% |
| 月末敏感 | `month_end_flag` = 1 |

重新运行后，Forecast 会用这一新的未来上下文重新计算轨迹。

## 需要避免的陷阱

如果协变量准备不当，可能反而降低结果质量。

应避免：

- 已知未来中变量留空；
- 不提供任何信息的常量变量；
- 未转换为可用值的自由文本；
- 间接包含未来目标的变量；
- 同一列中混用单位；
- 没有清晰假设而凭空捏造的未来值。

# 多序列

多序列允许在同一次分析中预测多个序列。Forecast 不必为每个对象单独启动一次预测，而是接收一个数据集，其中包含一列用于标识每个序列。

## 示例

一个文件可以包含多个门店的销量：

```text
date        magasin   ventes   promo
2026-05-01  paris    120      0
2026-05-01  lyon     92       1
2026-05-02  paris    132      1
2026-05-02  lyon     95       0
```

这里，`magasin` 是序列列。

Forecast 知道它需要预测 `paris` 和 `lyon` 的销量。

## 有什么用

当多个序列具有共同逻辑时，多序列很有用。

示例：

- 各门店的销量；
- 各餐厅的订单；
- 各服务器的流量；
- 各资产的价格；
- 各地区的事故数。

模型可以比单独预测利用更多信息，尤其是在各序列相似或共享上下文变量时。

## 每个序列的预测期

每个序列必须提供一致的时间结构。

如果预测期为 `31`，每个序列都必须有 31 个待预测的未来点。

示例：

```text
paris -> 31 行未来数据
lyon  -> 31 行未来数据
```

不一致的预测期会让对比变得困难，并可能阻塞模型。

## 在 Forecast 中查看

在界面中，用户可以选择要显示的序列。

随后场景可以：

- 针对某个具体序列查看；
- 针对多个序列查看；
- 与基准预测进行对比。

多序列并不改变 Forecast 的原理。它只是增加了一个维度：“当前正在预测的是哪个序列？”

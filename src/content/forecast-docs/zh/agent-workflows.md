# LLM 代理

LLM 代理可以把 Forecast 当作一个专用引擎来使用。它们的职责不止于读取文件和点击某个 tool：它们可以准备数据、在网上查找上下文、构建数据集、启动预测，并解释结果。

## 代理能做什么

代理可以在多个环节介入：

| 环节 | 代理的职责 |
| --- | --- |
| 准备 | 读取 Excel、CSV 或 JSON |
| 检索 | 在网上获取外部信息 |
| 数据集 | 创建或补全有用的列 |
| 启动 | 用正确的参数调用 `forecast` |
| 读取 | 使用 `forecast_read` 取回结果 |
| 场景 | 创建假设并重新运行模型 |
| 解释 | 概括趋势、不确定性、变量与局限 |

示例：对于一份金融预测，代理可以读取本地文件、查找近期市场上下文、生成诸如 `news_score` 或 `event_flag` 的列，然后启动 Forecast。

## 推荐工作流

代理应遵循以下顺序：

1. 理解用户的诉求；
2. 检查可用数据；
3. 识别要预测的目标；
4. 识别日期、频率和预测期；
5. 如有必要，查找或创建有用的上下文变量；
6. 检查未来行是否一致；
7. 选择一个兼容的模型；
8. 启动 `forecast`；
9. 用 `forecast_read` 复查结果；
10. 解释预测并提出有用的场景。

## 代理创建数据

如果用户要求或预测需要，代理可以创建数据。

示例：

- 根据日期添加一列 `weekend`；
- 创建 `month_end_flag`；
- 将网上事件转换为数值得分；
- 用天气假设填充未来区；
- 构建一个测试数据集以验证工作流；
- 将 Excel 文件转换为可用的 JSON。

代理必须始终说明它创建了哪些列以及为什么。

## 安全与质量规则

代理不得静默地凭空捏造重要数据。

如果它创建了一个变量，必须区分：

- 从文件中读取的数据；
- 从网上找到的数据；
- 计算得出的数据；
- 用于模拟的假设。

这种区分至关重要，以便用户知道什么是真实的、计算的、还是假设的。

## 斜杠命令

斜杠命令用于为代理和用户提供快速指引。

示例：

- `/forecast`：了解 Forecast 模块；
- `/forecast-predict`：准备并启动一份预测；
- `/forecast-dataset`：构建一个干净的数据集；
- `/forecast-scenarios`：创建有用的假设；
- `/forecast-cmd`：了解可用的 tools。

这些命令应当给出简短、清晰、可直接执行的流程。
